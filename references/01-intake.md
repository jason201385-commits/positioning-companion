# 01 · Intake 協定（七面向釐清）

> Intake 的目的不是填滿表格，而是取得足以判斷 `content`、`workflow`、`business-amplifier` 或 `pause` 的證據。不要預設每個人都要轉行、做內容或做產品。

## 0. 先選深度與資料規則

- `run_depth: quick`：取得具體痛點、已驗證能力、交付紅線、財務壓力與想改善結果；證據不足時標 `partial`。
- `run_depth: full`：補齊七面向、反向 gate、候選比較與最小驗證。

所有筆記標記來源：

- `[使用者原話]`：逐字或經本人確認的近義改寫。
- `[AI 建議]`：AI 根據原話做的暫定推論。
- `[待驗證]`：需要真實行為、受眾或現場資料。

薪資、個資、病患資料、客戶資料、公司機密等敏感資訊不要求貼入 AI。不想回答可跳過並標 `待補`；測試時先去識別化，保留人工覆核與回復方式。

## 1. 開場與回合規則

開場：

> 「我不急著替你選職業，想先搞懂你現在卡在哪、哪些東西其實值得保留。我會一次問一題；我的推論如果不像你，請直接糾正。」

規則：

1. 一回合預設一題，上一題沒落地不丟下一題。
2. 先說明問題用途，再問；錢與敏感資料不逼答。
3. 抽象回答要追：「上一次發生是什麼情況？給我一個畫面。」
4. 每完成一個 ★ 面向，先回述並請本人確認。未確認前只能標 `[AI 建議]`。
5. 本人糾正時直接修正，不為 AI 的判斷辯護。

## 2. 七面向

### ① 經濟狀況／風險承受

**用途**：判斷新方向的風險能否承受，不替本人做財務決定。

可問：

- 「這次改變如果沒有帶來新收入，生活會立刻出問題，還是仍可承擔？」
- 「不用講金額；現在是緊、普通，還是可承擔？」
- 「有哪些固定責任或現有收入不能被新實驗破壞？」
- 「你真實能投入多少時間？不用報理想版本。」

紅旗：

- 把內容或新產品當成短期現金急救。
- 新方向要求先放棄主要收入，卻沒有安全底或回復方案。
- AI 直接建議辭職、借錢或投資。

遇到紅旗時，合法輸出是 `gate_status: fail` 或 `mode: pause`；提供判斷框架，由本人決定。

### ② 學經歷／身分資產

**用途**：找已累積的可信度、場景理解與特殊交叉，不只看學歷職稱。

可問：

- 「你做過最久的一件事是什麼？看過哪些別人不一定看過的現場？」
- 「哪段經歷讓別人比較願意聽你？」
- 「你的背景裡，哪兩件事放在一起有點特別？」
- 「現有身分裡，哪一部分其實有效、你不想丟掉？」

追問具體故事，不把漂亮履歷自動等同適合公開內容或新事業。

### ③ 專業技能 ★

**用途**：盤點已存在、可交付或可教清楚的能力。

可問：

- 「朋友或同事卡住時，會來問你什麼？」
- 「上一次你比一般人更快或更穩地做完一件事，是什麼？」
- 「如果要教別人做到一個小結果，你最有把握教什麼？」
- 「你現在用 AI 是聊天、協助工作，還是能設計與驗收完整流程？」

追問：

- 區分「我想學」與「我已做過」。
- 區分「自己會」與「能讓別人成功重做」。
- 把整理、說人話、流程設計、品質檢查等隱性能力也記錄。

完成條件：至少一個具體事件，並由本人確認回述。

### ④ 興趣／燃料

**用途**：在幾個可行候選之間判斷願不願意繼續，不讓興趣壓過痛點、能力與安全條件。

可問：

- 「最近一次聊到停不下來的是什麼？」
- 「沒人付錢、沒人看，你仍會研究或動手做什麼？」
- 「這個興趣只有消費，還是你有作品、觀點或實作？」

只把興趣當候選燃料，不直接推成賽道。

### ⑤ 厭惡／想逃離什麼 ★

**用途**：取得痛點與交付硬紅線。

可問：

- 「現在工作或生活裡，最想擺脫哪個具體情況？」
- 「上一次遇到它是什麼畫面？」
- 「哪種做法讓你看不下去？如果換你做，會改哪裡？」
- 「什麼形式打死不做：露臉、即時回覆、陌生開發、長文、複雜維護，還是別的？」

把答案分成：

- 個人厭惡 → 交付硬紅線，踩到就淘汰。
- 市場厭惡 → 可能的品質縫隙，但仍需外部驗證。

完成條件：一個具體痛點場景 + 一條本人確認的硬紅線。

### ⑥ 當前現金流／現有價值

**用途**：判斷是要增加收入、改善本業，還是轉換主業；不要求精確金額。

可問：

- 「收入主要來自固定薪水、接案、生意，還是混合？」
- 「你希望新方向多一份收入、改善現有工作，還是換掉主收入？」
- 「現有客戶、流程或能力裡，哪一個最值得保留？」
- 「哪個收入來源最不能被測試破壞？」

若改善本業已能解題，優先保留 `business-amplifier`；不要把轉行當成進步的唯一形式。

### ⑦ 最大問題 ★

**用途**：判斷問題是方向、工作流、啟動障礙，還是目前沒有必要改變。

可問：

- 「如果現在只改善一件事，你會選什麼？」
- 「上一次卡在這是什麼情況？」
- 「你試過什麼？卡點在哪？」
- 「這是你一個人的問題、某個內部流程，還是一群外部受眾也有？」
- 「想到開始時，第一個『可是』是什麼？」

分類：

- **方向型**：外部受眾也反覆有此痛，可生 content／service 候選。
- **工作流型**：具體內部流程卡住，優先 workflow。
- **本業放大型**：本業有效，只需改善一個環節。
- **障礙型**：沒有時間、技能、權限或安全條件；先拆障礙，不急著定位。
- **暫停型**：沒有明確問題或改變意願，輸出 pause。

完成條件：具體場景 + 問題分類 + 本人確認。

## 3. 收斂前反向 gate

Intake 結束前逐一問：

1. 真的需要新方向嗎？
2. 完全不做內容，仍能驗證嗎？
3. 完全不做新產品，用人工服務、既有工具或範本仍能解題嗎？
4. 財務、安全、時間與權限條件允許最小測試嗎？
5. 哪個結果出現，就代表 AI 的建議錯了？

以上答案直接成為定位卡的 `gate_details`、`no_product_alternative` 與 `falsifiable_check`。

## 5. Intake 完成檢查

- [ ] 已記錄 run_depth？
- [ ] ③技能、⑤厭惡、⑦最大問題都有具體場景並經本人確認？
- [ ] 已分開 `[使用者原話]`、`[AI 建議]`、`[待驗證]`？
- [ ] 已記錄現況裡值得保留的本業資產？
- [ ] 已判斷方向型、工作流型、本業放大型、障礙型或暫停型？
- [ ] 已跑「需要新方向／內容／產品嗎」反向 gate？
- [ ] 已填 `no_product_alternative` 與 `falsifiable_check`？
- [ ] 財務、安全、權限與敏感資料紅線已處理？
- [ ] 公開存檔使用代號，沒有可識別個資？

完成後交棒 `02-choose-lane.md`。有缺口就標 `partial` 或 `pause`，不要硬推進。
