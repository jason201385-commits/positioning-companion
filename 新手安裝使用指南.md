# 🚀 新手安裝使用指南

> 先選「不用安裝」或「安裝成 Skill」。兩條路都能完成定位；安裝只是讓之後重複使用更方便。

---

## 先選 Quick 或 Full

- **Quick 快速草稿**：先找方向假設。證據不足時，定位卡會標 `gate_status: partial`。
- **Full 完整驗證**：補齊七面向、反向 gate、真實證據與最小實驗。

不用先猜哪個比較好。可以先 quick，再請 AI「補成 full」。本工具不宣稱固定完成時間。

---

## 路 A：完全不用安裝

1. 打開 ChatGPT、[Claude.ai](https://claude.ai)、Gemini 或其他可長對話的 AI。
2. 打開 [`一鍵定位-貼上任何AI.md`](一鍵定位-貼上任何AI.md)。
3. 整份複製，貼進新的 AI 對話。
4. 輸入「開始」。
5. 選 quick 或 full，再一題一題回答。

最後會得到 `positioning-card/v2`，其中會清楚標示：

- 你的原話與 AI 建議。
- 建議模式：內容、工作流、本業放大或暫停。
- gate 是否通過，以及缺什麼證據。
- 不做新產品的替代方案。
- 第一個真實證據與可推翻檢查。

這條路不需要 Claude Code，也不需要任何外部 Skill。

---

## 路 B：安裝成 Claude Code Skill

### Step 1｜安裝 Claude Code

可參考 Jason 的圖文教學：<https://claude.easyknowai.com/>。

產品方案與額度可能變動，請以 Claude 官方當下頁面為準；定位小書僮不要求特定付費方案。

### Step 2｜請 Claude 幫你安裝

把 repo 網址貼給 Claude Code：

```text
https://github.com/jason201385-commits/positioning-companion
幫我把這個 repo 安裝成 positioning-companion skill
```

### Step 3｜檢查最終路徑

資料夾名稱應為 `positioning-companion`，不是中文品牌名：

```text
~/.claude/skills/positioning-companion/SKILL.md
```

Windows 範例：

```text
C:\Users\<你>\.claude\skills\positioning-companion\SKILL.md
```

若你手動安裝：

1. 建立 `~/.claude/skills/positioning-companion/`。
2. 把 repo 內的 `SKILL.md`、`references/`、`runners/` 等檔案放進去。
3. 重新開啟 Claude Code。
4. 輸入「幫我定位」或「我需要做內容或產品嗎」。

## 不裝配套工具也能完成

定位小書僮本身已自足。遇到外部工具名稱時，一律把它當**可選資源**：

| 任務 | 純文字 fallback |
|---|---|
| 找真實受眾 | 直接列 10 個真名，記錄他們是否真的有這個痛、目前怎麼解 |
| 寫第一份內容 | 寫四行：對誰說／他卡在哪／我的證據／一個可做動作 |
| 驗證工作流 | 用紙筆或試算表記錄原流程、測試流程、例外與回復結果 |
| 驗證產品 | 先人工交付一次，觀察需求是否重複，不先寫完整產品 |

定位完成後，若你已決定要做內容、視覺或影片，才按需選裝：

| 可選工具 | 用途 |
|---|---|
| [social-post](https://github.com/Hao0321/claude-skill-social-post) | 協助內容草稿與發佈流程 |
| [ai-media-generator](https://github.com/Hao0321/ai-media-generator) | 協助視覺與影音提示方向 |
| [video-autopilot-kit](https://github.com/Hao0321/video-autopilot-kit) | 協助影片後製流程 |
| [code-cleanup](https://github.com/Hao0321/claude-skill-code-cleanup) | 協助清理 Skill／repo |

沒有這些工具，不影響完成定位。

## 常見卡點

| 遇到 | 處理方式 |
|---|---|
| AI 太快下結論 | 回覆：「這是 quick 草稿，請補成 full，並標出缺的證據。」 |
| AI 把推論寫成我的原話 | 回覆：「請把 `[使用者原話]` 與 `[AI 建議]` 分開；未確認的標 `[待驗證]`。」 |
| 我不想做內容 | 回覆：「請跑內容反向 gate，改看 workflow、business-amplifier 或 pause。」 |
| 我不想做產品 | 回覆：「請先填 `no_product_alternative`，用人工服務或既有工具驗證。」 |
| 安裝路徑錯誤 | 確認最終檔案是 `~/.claude/skills/positioning-companion/SKILL.md` |
| 畫面或錯誤訊息看不懂 | 截圖時先遮住 API key、token、客戶資料與公司機密，再請 AI 解讀 |
